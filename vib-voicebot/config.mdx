# VIB Voice Agent — Configuration Guide

This page documents the exact production configuration of the VIB Voice Agent. Use it as the source of truth when reviewing changes, onboarding new operators, or replicating the agent in another workspace.

| Setting | Value |
|---------|-------|

| **Engine** | `Simple Prompt` |
| **Region** | 🇻🇳 Việt Nam |

## Call behavior

| Field | Value |
|-------|-------|
| Max Duration | `5` minutes |
| Begin Delay | `1000` ms |
| End After Silence | `10` s |
| AI Speaks First | `on` |
| Silence Timeout (conversation) | `15` s |
| Max Follow-ups | `2` |

The agent always opens the call — there is no IVR step in front of it.

## Tools

Four production tools are enabled:

| Tool | Purpose |
|------|---------|
| `extract_and_validate_phone` | Pull a phone number out of the transcript, normalize to 10 digits, validate format. Called every time the customer speaks digits. |
| `verify_phone` | Check the collected phone number against the VIB CRM. On failure the agent re-confirms once, then ends the call. |
| `norm_address` | Normalize a free-form address into structured street / ward / district / city. Called after every address answer; agent reads back the normalized form for confirmation. |
| `end_call` | End the call gracefully. Guarded by a two-condition rule (see prompt, section 5 / "QUY TẮC GỌI end_call"). |

## Call variables

Seeded by the outbound campaign or API trigger. The prompt branches on these:

| Variable | Used for |
|----------|----------|
| `predicted_gender` | Picks `anh` / `chị` / `bạn` form of address |
| `UF_CRM_1489666359` | CRM gender (30 = male, 31 = female) |
| `inbound` | `true` when the customer is calling back (vs. outbound) |
| `collecting_info` | `true` when the customer is continuing a previous partially-completed call |
| `fullName`, `phoneNumber`, `UF_CRM_*` | Any field already on file — the prompt skips asking for fields that are already present |

## Call flow (summary)

```
BƯỚC 1 – Greeting (with inbound / collecting_info branches)
      │
      ▼
BƯỚC 2 – Callback reschedule  (if customer is busy)
      │
      ▼
BƯỚC 3 – Introduce purpose + collect phone number
      │
      ▼
extract_and_validate_phone → verify_phone
      │
      ├── success ─▶ Collect basic fields (name, home address)
      │                  │
      │                  ▼
      │             Ask occupation
      │              ├── student  ─▶ Student fields (school name, year, address)
      │              └── employed ─▶ Job fields (status, income, company, address, tenure)
      │                  │
      │                  ▼
      │             Common fields (education, reference phone, card delivery address, security Q)
      │                  │
      │                  ▼
      │             End call
      │
      └── failure ─▶ Re-confirm once ─▶ end_call with "not a match" message
```

## Guardrails (non-negotiable)

From the prompt, sections 1 & 2:

- Vietnamese only.
- Never sell, negotiate, or add content outside the script.
- Never ask for information already captured — even after a silence retry.
- Never ask "or" questions ("A hay B").
- Never say the agent is a chatbot / AI unless the customer directly asks.
- After a refusal, handle one objection, then end the call politely.
- `end_call` only fires when (a) there is an explicit reason to end AND (b) the agent has already said goodbye AND the customer has responded.
