# VIB Voice Agent — Test Scripts

Run this checklist before publishing any new version of the VIB Voice Agent. Every line is phrased as what the **tester speaks** (as the customer) or what **the agent should do**. Deviations from the expected behavior are rollback signals.

Use the **Preview → Web Call** tab for rapid iteration, then repeat the critical paths on a real **Phone Call** before publishing.

## Seed data for Preview

Before every test, seed these Call Variables in the Preview tab so the agent behaves as in production:

| Variable | Value |
|----------|-------|
| `phoneNumber` | A phone number known to exist in the VIB CRM for verification success tests (or a deliberately mismatched one for failure tests). |
| `predicted_gender` | `male` / `female` / empty to test fallback. |
| `inbound` | `false` for outbound (default) / `true` for return-call tests. |
| `collecting_info` | `false` first-time / `true` for "continuation" tests. |
| `fullName`, `UF_CRM_*` | Populate a subset to test that the agent skips fields already on file. |

---

## Test 1 — Happy path (outbound, employed customer)

**Seed:** `predicted_gender=male`, `inbound=false`, `collecting_info=false`, no fields pre-filled.

| # | Tester speaks | Expected agent behavior |
|---|---------------|-------------------------|
| 1 | *(call connects)* | Agent opens: greets, introduces self as "Vi" from VIB, asks "Anh có tiện nghe máy không ạ?" |
| 2 | "Được." | Agent explains call purpose (complete credit-card application), starts with phone-number confirmation. |
| 3 | Speak a valid registered phone number | Agent calls `extract_and_validate_phone`, reads back in `0123-456-789` format, asks to confirm. |
| 4 | "Đúng." | Agent calls `verify_phone`, gets success, moves to basic fields. |
| 5 | Answer full name | Agent confirms, asks for house number + street. |
| 6 | Give a full address | Agent calls `norm_address`, reads back normalized address, asks to confirm. |
| 7 | "Đúng rồi." | Agent asks occupation. |
| 8 | "Em đi làm công ty." | Agent routes to employment fields — status, income, company name, company address, tenure. |
| 9 | Answer each field | Agent never asks a field that was already seeded. |
| 10 | Answer education level, reference phone, card delivery preference, primary-school security answer | Agent validates reference phone via `extract_and_validate_phone` (must differ from primary). Card delivery only offers home vs work (not a third option). |
| 11 | *(silence after last field)* | Agent thanks customer, asks them to check email, says goodbye, waits for response, then calls `end_call`. |

**Pass criteria:** no duplicate questions, no "hoặc/hay" phrasing, `norm_address` called after every address, call ends via `end_call` (not silence timeout).

---

## Test 2 — Happy path (student)

Same as Test 1 but at step 8 answer "Em là sinh viên."

**Expected:** agent routes to the student fields (school name + start year combined into one question, school address), skips the job fields entirely, then proceeds to common fields (education, reference phone, card delivery, security question).

---

## Test 3 — Inbound continuation

**Seed:** `inbound=true`, `collecting_info=true`, `fullName` + `phoneNumber` already set.

**Expected opening:** "Chào bạn, cảm ơn bạn đã liên hệ lại VIB. Mình có thể tiếp tục hoàn tất hồ sơ mở thẻ tín dụng VIB đang dang dở không ạ?"

Agent must not ask for `fullName` or `phoneNumber` again. Should jump straight to the first still-missing field.

---

## Test 4 — Customer is busy

**Seed:** outbound, no variables.

| # | Tester | Expected agent |
|---|--------|---------------|
| 1 | *(call connects)* | Greeting. |
| 2 | "Đang bận, lát gọi lại được không?" | Step 2 — asks when is a better time. |
| 3 | "3 giờ chiều nay." | Agent confirms time, thanks, ends the call. |
| 4 | — | `end_call` fires. |

**Pass criteria:** no attempt to push through to data collection; confirmation is spoken; no hang-up without saying goodbye.

---

## Test 5 — Refusal (first objection handled)

| # | Tester | Expected agent |
|---|--------|---------------|
| 1 | *(after greeting)* | Agent explains purpose. |
| 2 | "Không, không có nhu cầu." | Agent handles objection: explains customer registered for VIB card, offers to help complete the application. |
| 3 | "Không, thôi." | Agent thanks, says goodbye, calls `end_call`. |

**Fail case to avoid:** agent pushes a second objection (only one is allowed).

---

## Test 6 — Security objection

| # | Tester | Expected agent |
|---|--------|---------------|
| 1 | "Tôi không tin, sao tôi biết đây là VIB?" | Agent reassures about security compliance + confirms the outbound number is registered to VIB. |
| 2 | "Vẫn không muốn." | Agent thanks, ends the call. |

---

## Test 7 — Phone-verification failure

**Seed:** use a number NOT in the CRM.

| # | Tester | Expected agent |
|---|--------|---------------|
| 1 | Give the mismatched number | `verify_phone` returns failure (attempt 1). |
| 2 | Agent reads the number back, asks to confirm | Tester: "Đúng." |
| 3 | `verify_phone` called again, fails again | Agent says "Số điện thoại không khớp với hệ thống. Chúng tôi sẽ liên hệ lại sau." |
| 4 | — | `end_call` fires. Agent must not ask for any other fields. |

---

## Test 8 — Silence handling

| # | Tester | Expected agent |
|---|--------|---------------|
| 1 | *(stay silent after greeting)* | After the `End After Silence` window, agent re-prompts once. |
| 2 | *(stay silent again)* | Agent re-prompts up to `Max Follow-ups` (= 2). |
| 3 | *(stay silent)* | Call ends cleanly; disconnection reason = `silence_timeout`. |

---

## Test 9 — "giống địa chỉ trên"

At the company-address step, answer "giống địa chỉ nhà." Agent must confirm once (since home address was already given) and proceed. If home address was NOT given, agent must ask again.

---

## Test 10 — Request to speak to a human

At any point: "Cho tôi gặp nhân viên thật." The agent must acknowledge, say a VIB employee will contact the customer shortly, say goodbye, and end the call. It must NOT try to transfer during the test (there is no human queue wired in production for this POC).

---

## Test 11 — Carrier edge cases (phone-only)

These cannot be tested on Web Call — run them on a real phone:

| Scenario | Expected |
|----------|----------|
| Voicemail picks up | `detect_voicemail` triggers; agent leaves the configured message and hangs up. |
| Customer hangs up mid-sentence | Call record shows disconnection reason `human_hangup`; partial fields are still saved. |
| Poor audio / 3G noise | STT errors are high; the agent falls back to "xin phép xác nhận lại" flow. |
| Customer says "chào em" mid-call | Agent treats this as a greeting, NOT a goodbye. Does not call `end_call`. |

---

## Regression checklist before every publish

- [ ] Happy path — employed (Test 1) completes with `end_call`.
- [ ] Happy path — student (Test 2) routes correctly.
- [ ] Inbound continuation (Test 3) skips already-filled fields.
- [ ] Callback reschedule (Test 4) ends cleanly with time confirmed.
- [ ] First-refusal objection handled, second refusal ends the call (Test 5).
- [ ] Phone-verification failure ends after 2 attempts (Test 7).
- [ ] Silence path ends on `silence_timeout` after 2 follow-ups (Test 8).
- [ ] At least one real phone call completed on the new version without errors.
