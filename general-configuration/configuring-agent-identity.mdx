Every Chat Agent has three identity fields — **Name**, **Description**, and **Role** — on the **Basic** tab, under the **General Settings** sub-tab in the left sidebar. They control what the agent is called, how your team tells agents apart, and how the model behaves. **Name** and **Description** are required to save; **Role** is optional but almost always worth writing.

![Basic tab showing agent identity fields — Name, Description, and Role](/screenshots/chatbot-basic-settings.png)

<Note>
Changes here auto-save into your draft as you type. You need the draft edit lock (**Start Editing**) to change anything, and auto-saved changes only reach the live agent once you publish — see [Publishing and Versioning](/getting-started/publishing-and-versioning).
</Note>

## Identity Fields

| Setting | Default | When to change |
|---------|---------|----------------|
| **Name** | Empty (required) | This is the agent's display name — it appears in the chat widget header, the browser tab/page title, and any Open Graph or Twitter Card preview if you link directly to the agent's standalone chat page. It's also available elsewhere in the product as the `{{AI agent name}}` template variable (for example, in conversation-feedback thank-you messages), so renaming an agent updates every message that references it. Maximum 50 characters. Keep it short and recognizable — "Sales Assistant" or "Booking Concierge" reads better in a narrow widget header than a full product name. |
| **Description** | Empty (required) | Primarily a team-facing label — it's what shows up in agent-picker dropdowns and list views so you can tell agents apart once you have more than a couple. It also becomes the page meta description (and Open Graph/Twitter description) if you use the standalone full-page chat URL for this agent, so write it in plain, accurate language rather than internal shorthand if that channel is in play. Maximum 500 characters, but a one-sentence summary of what the agent does and who it's for is normally enough. |
| **Role** | Empty (optional) | A rich-text field (bold, italic, headings, lists, links) inserted into the system prompt under its own `## Role` section — separate from, and alongside, the **Objectives** field (the agent's instructions) and the team-level Business Description and Brand Voice. Leaving it empty is fine for a narrowly-scoped agent whose Objectives already say everything needed; write one when you want a persona that holds up across many different objectives — tone, boundaries, what it should never claim to be. Think "who is this agent" (Role) vs. "what should it accomplish right now" (Objectives), and keep the two from duplicating each other. |

<Tip>
Role and Objectives are easy to blur together. If you find yourself repeating the same behavioral instruction in both, it belongs in Role — it applies everywhere the agent talks, while Objectives is where you put what's specific to this agent's job.
</Tip>
