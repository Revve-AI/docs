Open your chat agent, select the **Appearance** tab, then **Content** in the left sidebar. This is where you control what a visitor sees the instant the chat widget opens, before they've typed a word: an optional **Welcome Messages** greeting and up to four **Conversation Starters** buttons.

![Welcome Messages and Conversation Starters on the Appearance Content tab](/screenshots/website-content.png)

Like the rest of a chat agent's settings, this tab auto-saves into your draft as you type — there's no page-level Save button. You need the draft edit lock (**Start Editing**) to change anything, and a saved change only reaches visitors once you publish — see [Publishing and Versioning](/getting-started/publishing-and-versioning).

## Welcome Messages

Welcome messages are assistant-style bubbles that appear automatically when the chat opens — before the visitor sends anything. They set the tone and tell visitors what the agent can help with.

| Setting | Default | When to change |
|---------|---------|-----------------|
| **Welcome Messages** | None | Click **Add welcome message** to add a bubble; click it again to queue a second, third, and so on — there's no fixed cap on how many you can add. Each entry sends as its own separate message, so two short lines read more like a natural greeting than one dense paragraph. Leave it empty if you'd rather let **Conversation Starters** initiate the conversation instead. |
| **Welcome messages location** | **Top** | The radio-card choice next to the message list. **Top** puts the greeting above any conversation, which is right for most agents since it's the first thing visible. Switch to **Bottom** if your widget layout puts the input field near the top and you want the greeting to sit closer to where the visitor is about to type. |

<Tip>
Keep it to one or two messages. A long welcome sequence delays the moment a visitor can actually type, and it reads as a wall of text rather than a greeting.
</Tip>

## Conversation Starters

Conversation starters are clickable buttons in the widget. Clicking one sends its text as the visitor's first message, jumping straight into that topic.

| Setting | Default | When to change |
|---------|---------|-----------------|
| **Conversation Starters (up to 4)** | None | Click **Add starter** for each one; the button disables once you have four — this cap is fixed in the UI, not editable. Removing the last remaining starter clears its text rather than deleting the row, so there's always at least one input to fill in. Write each one from the visitor's perspective ("I'd like to see pricing") and cover your top 3-4 contact reasons rather than trying to anticipate everything. |
| **Shuffle Conversation Starters** | Off | When on, the display order randomizes each time the widget opens or the page reloads. Turn it on once you have several starters and want them to get roughly equal exposure instead of the first one always winning the click. |

There's no enforced character limit on starter text in either the form or the underlying schema, but starters render as buttons — long text wraps or crowds the widget. Aim under 50 characters as a practical guideline, not a hard rule.
